Figure 1. Tofu is organised as five top-down layers. Entry ingests requests from a web UI, a group-chat bot, or a headless backend API that lets third-party CLIs embed Tofu as an execution harness. Orchestration offers three progressively richer agent-loop patterns—single-agent ReAct, a Planner–Worker–Critic triad, and a multi-agent swarm. Context Engineering is a cross-cutting layer that prepares each LLM call with prompt assembly, three-layer compaction, persistent memory, and message reconstruction. Tools & Extensions routes every tool call through a unified bus to four capability families and stays open to external protocols. LLM Dispatch abstracts a fleet of keys and providers behind a slot-based scheduler with auto-discovery, fallback, and self-healing.
